# v1.1.1

Version 1.1.1 introduces significant architectural changes to the Event Bus and adds new utilities for application infrastructure.

## Key Changes

### Topic-Only Event Bus (Breaking)
The `EventBus` has been completely redesigned into a strictly topic-based API, inspired by Kafka.
- **Strong Typing**: Every `EventTopic<T>` is associated with a specific event type.
- **Topic Registration**: Topics are declared upfront via a DSL, preventing accidental usage of unregistered channels.
- **DSL Improvements**: Added bus-wide defaults for replay, buffer capacity, and error handling that can be overridden per topic.

### Event Bus Lifecycle
The `EventBus` and its contained `EventTopic`s now implement the `Disposable` interface. Disposing the bus automatically and atomically disposes all active topics and subscriptions.

### Command-Line Argument Parsing
Introduced `ProgramArguments` for thread-safe parsing and access to command-line arguments, supporting both flags and key-value pairs.

### Application Environments
Added the `ApplicationMode` enum to manage different environments like `DEV`, `STAGE`, and `PROD` with built-in serialization and validation.

## Infrastructure Updates
- **Kotlin 2.4.0**: Updated the Kotlin compiler to the latest version.
- **Gradle 9.6.1**: Bumped the Gradle Wrapper for improved build performance.
